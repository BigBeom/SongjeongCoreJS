# 송정 Core JS 스터디 2주차

### 실행 컨텍스트

- 실행할 코드에 제공할 환경 정보들을 모아놓은 객체
- 하나의 실행 컨텍스트를 구성하는 방법에는
    
    전역공간, eval()함수, 함수 등
    
    →간단히 함수를 실행하는 것 뿐
    

- js파일이 열리는 순간 전역 컨텍스트 활성화
- 함수 outer(예시)가 있으면 전역 컨텍스트와 관련된 코드의 실행을 일시 중단하고 outer 실행 컨텍스트와 관련된 코드, 즉 outer 함수 내부의 코드들을 순차로 진행
- 어떤 실행 컨텍스트가 활성화 될 때 자바스크립트 엔진은 해당 컨텍스트에 관련된 코드들을 실행하는데 필요한 환경 정보들을 수집해서 실행 컨텍스트 객체 저장합니다. 이 객체는 자바스크립트 엔진이 활용할 목적으로 생설할 뿐 개발자가 코드를 통해 확인할 수는 없습니다.

1. Variable Environment

현재 컨텍스트 내의 식별자들에 대한 정보 + 외부 환경 정보. 선언 시점의 lexical의 스냅샷으로, 변경사항은 반영 되지 않음.

1. Lexical Environment

처음에는 variable과 같지만 변경사항이 실시간으로 반영됨

1. This Binding:

식별자가 바라봐야 할 대상 객체

### Variable Environment

- 실행 컨텍스트 생성할떄, Variable Environment:에 정보를 먼저 담은 다음 복사해 Lexical Environment를 만들고 활용함
- 내부는 environmentRecord와 outer-EnvironmentReference로 구성돼 있음

### Lexical Environment

- 사전적 환경.컨텍스트를 구성하는 환경 정보들을 사전에서 접하는 느낌으로 모아놓은 것

### environmentRecord

컨텍스트와 관련된 코드의 식별자 정보들이 순서대로 저장됨(매개변수 이름, 함수선언, 변수명 등)

코드가 실행되기 전에 js 엔진은 이미 해당 환경에 속한 코드의 변수명을 모두 알고 있는 셈.

“마치” 식별자들을 최상단으로 끌어올려놓은 다음 실제 코드를 실행한다.

여기서 발생한 개념인 “끌어올리다” = 호이스팅

### 전역변수와 지역변수

전역변수: 전역 공간에서 선언한 변수

지역변수: 함수 내부에서 선언한 변수

### 함수 선언문

function 정의부만 존재하고 별도의 할당 명령이 없는 것

전체를 호이스팅

### 함수 표현식

정의한 function을 별도의 변수에 할당하는 것

변수 선언부만 호이스팅

### 스코프

식별자에 대한 유효범위

경계 A의 외부에서 선언한 변수는 A 내,외부에서 접근 가능하지만,

A내부에서 선언한 변수는 오직 A 내부에서만 접근 가능

- 스코프 체인

스코프의 구조적 특성

식별자의 유효범위를 안에서부터 바깥으로 차례로 검색하는 것(outer-EnvironmentReference)

여러 스코프에서 동일한 식별자를 선언한 경우에는 무조건 스코프 체인 상에서 가장 먼저 발견된 식별자에만 접근 가능
